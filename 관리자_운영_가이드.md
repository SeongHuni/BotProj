# 관리자 운영 가이드

이 문서는 관리자가 Discord whitelist 봇을 실제 서버에 등록하고, 사용자가 봇과 1대1 DM으로 whitelist를 신청하게 만드는 절차입니다.

## 1. 핵심 결정 3개

1. 사용자 편의성을 위해 기본 인증 방식은 봇 DM에 Minecraft 닉네임만 보내는 방식입니다.
2. 서버 채널 `/verify 닉네임`도 계속 지원하지만, `VERIFY_CHANNEL_ID`는 채널 명령 제한용 보조 설정입니다.
3. RCON은 같은 VM의 `127.0.0.1:25575`로만 연결하고 Google Cloud 방화벽에는 공개하지 않습니다.

## 2. Discord Developer Portal 설정

1. Discord Developer Portal에서 New Application을 생성합니다.
2. Bot 메뉴에서 봇을 생성하고 Token을 발급합니다.
3. Bot 설정에서 `Message Content Intent`를 켭니다. DM으로 닉네임만 받기 위해 필요합니다.
4. OAuth2 URL Generator에서 `bot`, `applications.commands`를 선택합니다.
5. Bot Permissions에서 `View Channels`, `Send Messages`, `Read Message History`, `Use Slash Commands`를 선택합니다.
6. 인증 역할 자동 부여를 사용할 경우 `Manage Roles`도 추가합니다.
7. 생성된 초대 URL로 Discord 서버에 봇을 초대합니다.

## 3. 사용자 인증 흐름

사용자는 Discord 서버에 들어온 뒤 봇에게 직접 DM을 보냅니다.

```text
Steve
```

또는:

```text
/verify Steve
```

봇은 닉네임 형식과 중복 등록을 확인한 뒤 RCON으로 `whitelist add Steve`를 실행합니다. 성공하면 사용자 DM으로 서버 주소를 안내합니다.

## 4. Discord 채널 준비

권장 채널:

```text
#공지
#서버-규칙
#마크-인증
#서버-주소
#운영-로그
```

`#마크-인증`에는 "봇에게 DM으로 닉네임을 보내세요"라는 안내문을 고정합니다. `#서버-주소`는 인증 완료 역할을 가진 사용자만 볼 수 있게 권한을 제한합니다.

## 5. Minecraft 서버 설정

`/home/seonghun9290/minecraft/server.properties`에 다음을 설정합니다.

```properties
online-mode=true
white-list=true
enforce-whitelist=true
enable-rcon=true
rcon.port=25575
rcon.password=강력한_RCON_비밀번호
```

Google Cloud 방화벽은 `tcp:25565`만 외부 공개합니다. `tcp:25575`, `tcp:8080`은 열지 않습니다.

## 6. 봇 설치

```bash
cd /home/seonghun9290/discord-whitelist-bot
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
nano .env
```

`.env` 필수 값:

```env
DISCORD_TOKEN=디스코드_봇_토큰
DISCORD_GUILD_ID=디스코드_서버_ID
ALLOW_DM_VERIFY=true
RCON_HOST=127.0.0.1
RCON_PORT=25575
RCON_PASSWORD=server.properties와_동일한_비밀번호
MINECRAFT_SERVER_IP=서버_접속_IP
MINECRAFT_SERVER_PORT=25565
```

서버 채널 `/verify`를 특정 채널에서만 허용하려면:

```env
VERIFY_CHANNEL_ID=마크_인증_채널_ID
```

승인 모드 사용 시:

```env
REQUIRE_ADMIN_APPROVAL=true
APPROVAL_CHANNEL_ID=운영진_승인_채널_ID
```

인증 역할 자동 부여 사용 시:

```env
VERIFIED_ROLE_ID=인증됨_역할_ID
```

## 7. 실행

```bash
source .venv/bin/activate
python bot.py
```

DM 테스트:

```text
봇에게 DM → Steve
```

서버 채널 테스트:

```text
/verify Steve
```

## 8. systemd 등록

`/etc/systemd/system/discord-whitelist-bot.service`:

```ini
[Unit]
Description=Discord Whitelist Bot
After=network.target

[Service]
User=seonghun9290
WorkingDirectory=/home/seonghun9290/discord-whitelist-bot
ExecStart=/home/seonghun9290/discord-whitelist-bot/.venv/bin/python /home/seonghun9290/discord-whitelist-bot/bot.py
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

적용:

```bash
sudo systemctl daemon-reload
sudo systemctl enable discord-whitelist-bot
sudo systemctl start discord-whitelist-bot
sudo systemctl status discord-whitelist-bot
```

로그 확인:

```bash
journalctl -u discord-whitelist-bot -f
tail -f /home/seonghun9290/discord-whitelist-bot/logs/bot.log
```

## 9. 사용자 안내문

`#마크-인증` 채널에 고정할 문구:

```text
[마인크래프트 서버 인증 방법]

1. Minecraft Java Edition 닉네임을 확인합니다.
2. 이 봇에게 1대1 DM으로 닉네임만 보냅니다.

예시:
Steve

3. 등록 완료 DM이 오면 Minecraft 서버에 접속합니다.
4. 서버 주소는 인증 완료 후 #서버-주소 채널에서도 확인할 수 있습니다.

주의:
- 닉네임은 3~16자의 영문, 숫자, 언더스코어만 가능합니다.
- 오타가 있으면 접속할 수 없습니다.
- 다른 사람의 닉네임을 등록하지 마세요.
```

## 10. 검증 체크리스트

- [ ] 봇에게 DM으로 `Steve`를 보내면 등록 완료 메시지가 온다.
- [ ] Minecraft 콘솔에서 `whitelist list` 실행 시 닉네임이 보인다.
- [ ] 같은 Discord 계정으로 다른 닉네임을 등록하면 차단된다.
- [ ] `ab`, `nickname!`, `한글닉네임`은 형식 오류로 차단된다.
- [ ] Minecraft 서버를 끈 상태에서 DM 등록을 시도하면 연결 실패 메시지와 로그가 남는다.
- [ ] 인증 역할을 설정한 경우 등록 완료 후 역할이 부여된다.
